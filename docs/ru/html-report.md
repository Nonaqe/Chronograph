# HTML-отчёт

[← Оглавление](README.md) · [English](../en/html-report.md)

`chronograph report` рендерит один **self‑contained** `report.html`: весь CSS инлайнится, все данные встроены как JSON, графики — server‑side SVG, сгенерированный в Rust. **Ноль внешних запросов** — ни CDN, ни шрифтов, ни скриптов, подгружаемых при просмотре. Файл можно отправить почтой, закоммитить или разместить где угодно.

```bash
chronograph report /путь/к/репо --out report.html
```

## Что внутри

| Секция | Содержимое |
|---|---|
| **Overview** | Метаданные репозитория: версия движка, хэш конфига, `head_sha`, счётчики коммитов/файлов/авторов. |
| **Hotspots** | **Squarified treemap**, где площадь = сложность, цвет = churn (бледный → тёмно‑красный). |
| **Coupling** | Таблица топ change‑coupling пар (support и ratio). |
| **Knowledge** | По каждому файлу — bus factor и доля топ‑владельца, авторы анонимны по умолчанию. |

Treemap Hotspots рисуется как SVG прямо в Rust (раскладка squarified / Bruls). Вход сортируется по `(complexity desc, path asc)` до раскладки, поэтому ячейки равной площади раскладываются воспроизводимо.

## Детерминизм

Два прогона на одном коммите дают **байт‑идентичный** `report.html`. Обеспечивается:

- Детерминированной сериализацией JSON (сортировка, фиксированный формат float).
- Server‑side SVG (браузерный движок раскладки не участвует).
- В файле только детерминированные метаданные — `engine_version`, `config_hash`, `head_sha`. Wall‑clock `analyzed_at` **не** встроен в ломающем байт‑идентичность виде.

Тест воспроизводимости проверяет это байт‑в‑байт.

## Опции

| Флаг | Значение |
|---|---|
| `--out <FILE>` | Путь вывода (по умолчанию `report.html`). |
| `--db <FILE>` | Расположение кэша. |
| `--exclude <GLOB>` | Исключить пути (можно повторять). |
| `--blame-budget <N>` | Бюджет blame на файл для секции knowledge (по умолчанию 10 000 000; `0` = безлимит). |

## Отчёт vs. web UI

HTML‑отчёт **статичен и self‑contained** — идеален для CI‑артефактов и обмена. Для **интерактивного** исследования (force‑граф, zoomable treemaps, ползунок timeline, анимация роста) используйте [Web UI](web-ui.md) с JSON от `chronograph export`. Отчёт делает ставку на переносимость; web UI — на интерактивность.
