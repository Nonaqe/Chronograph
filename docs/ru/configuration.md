# Конфигурация

[← Оглавление](README.md) · [English](../en/configuration.md)

Chronograph предпочитает **явные конфигурируемые пороги** зашитым константам. Здесь собраны все «ручки» и их дефолты.

## Glob-исключения — `--exclude`

Исключите vendored‑ или generated‑пути из анализа. Можно повторять.

```bash
chronograph hotspots . \
  --exclude "vendor/**" \
  --exclude "**/*.min.js" \
  --exclude "third_party/**"
```

Исключённые пути вообще не попадают в анализ, поэтому не могут исказить churn, coupling, hotspots или knowledge.

## Окна churn

Churn считается по трём скользящим окнам с дефолтами **30 / 90 / 365 дней**, взятыми прямо из ТЗ. Окна отсчитываются от `anchor = max(committed_at)` (последняя активность в истории), а **не** от wall‑clock «сейчас» — именно это делает вывод воспроизводимым. Дефолты конфигурируемы в библиотеке (`ChurnConfig`); CLI использует значения из ТЗ.

## Support для coupling — `--min-support`

```bash
chronograph coupling . --min-support 5     # по умолчанию
```

Паре файлов нужно как минимум `min_support` совместных коммитов, чтобы попасть в вывод. Дефолт **5** отсекает случайные разовые со‑изменения. Понизьте, чтобы вытащить больше (шумных) пар; повысьте — для только сильнейших сцеплений.

## Механические коммиты

Флаг: `--mechanical-max-files`

```bash
chronograph analyze . --mechanical-max-files 50
```

Коммит, тронувший больше **N** файлов, помечается как *механический* (массовый rename, прогон форматирования, дроп vendored, релизный бамп). Механические коммиты искажают churn и coupling, поэтому обе метрики **исключают их по умолчанию**, когда эвристика включена.

> **По умолчанию выключено** (порога нет). Хорошее значение зависит от репозитория, а ТЗ его не фиксирует — Chronograph не выдумывает магическое число. Включайте явно, когда в репозитории есть массовые коммиты, загрязняющие сигнал. Флаг задаётся на этапе `analyze` (это свойство ingestion).

## Бюджет blame

Флаг: `--blame-budget`

```bash
chronograph knowledge . --blame-budget 10000000   # по умолчанию
chronograph age .       --blame-budget 0          # безлимит
```

Blame питает метрики **knowledge** и **code age**. Blame одного патологически большого, часто переписываемого файла (например, многомегабайтного генерируемого `CHANGELOG` с сотнями ревизий) неделим и может занять десятки минут; кэш не спасает, потому что такой файл инвалидируется почти каждым коммитом.

Бюджет ограничивает стоимость на файл как:

```
cost = ревизии × суммарно_добавленных_строк
```

Файлы, чья стоимость превышает бюджет, **пропускаются и указываются** (в списке `blame_skips` / сводке пропусков CLI / отчёте), никогда не выбрасываются молча.

- **По умолчанию 10 000 000.** Выбрано по данным: нормальный код имеет `cost < 1M`; патологические генерируемые гиганты превышают 30M. Дефолт режет разрыв с запасом ~×10 в обе стороны. На репо вроде ripgrep не отсекает ничего (max ≈ 1M).
- **`0` = безлимит** — блеймить всё, как бы медленно ни было.

`--blame-budget` применяется к `knowledge`, `age`, `report` и `export`.

## Имена авторов — `--show-names`

```bash
chronograph knowledge . --show-names
chronograph export . --show-names --out chronograph.json
```

По умолчанию метрика knowledge и JSON‑экспорт **анонимизируют** авторов как `Author #N`. `--show-names` показывает реальные имена. Выключено по умолчанию — продуктовое требование: карта знаний это *риск концентрации*, а не реестр заслуг/вины.

## Расположение кэша — `--db`

```bash
chronograph report . --db /tmp/chronograph-cache.duckdb
```

Перенести кэш DuckDB. По умолчанию `<repo>/.chronograph/cache.duckdb`.

## Полный пересчёт — `--no-incremental`

```bash
chronograph analyze . --no-incremental
```

Форсировать полный пересчёт вместо обработки только новых коммитов. Полезно после апгрейда движка или при подозрении на устаревший кэш. (Удаление `.chronograph/` даёт тот же результат.)

## Дефолты кратко

| Параметр | По умолчанию | Флаг |
|---|---|---|
| Путь кэша | `<repo>/.chronograph/cache.duckdb` | `--db` |
| Окна churn | 30 / 90 / 365 дней | *(библиотека)* |
| Min support для coupling | 5 | `--min-support` |
| Порог механических | выкл | `--mechanical-max-files` |
| Бюджет blame | 10 000 000 | `--blame-budget` |
| Имена авторов | анонимно | `--show-names` |
| Топ‑N строк | 20 | `--top` |
| Инкрементально | вкл | `--no-incremental` |
